# PRD: DART 재무분석기

## 개요

DART(전자공시시스템) Open API를 이용해 회사명으로 검색한 3개 회사의 재무제표를 비교하고, 결과를 엑셀 파일로 저장하는 개인용 CLI 도구.

## 배경 / 문제 정의

공인회계사가 여러 회사의 재무제표를 비교 분석할 때, DART 웹사이트에서 회사마다 개별적으로 조회하고 수작업으로 엑셀에 옮겨 정리하는 과정이 반복적이고 번거롭다. 회사명만 입력하면 자동으로 조회·비교·엑셀 저장까지 처리하는 도구가 필요하다.

## 목표

- 회사명 입력만으로 DART 고유번호(corp_code)를 자동 매칭
- 3개 회사의 최근 3개년 재무제표를 한 번에 조회
- 핵심 재무 항목과 주요 비율을 자동 계산해 비교
- 결과를 바로 활용 가능한 엑셀 파일로 저장

## 대상 사용자

재무제표를 실무적으로 비교·분석하는 개인 사용자 (회계사, 애널리스트 등). 다중 사용자, 배포용 서비스가 아닌 개인 로컬 실행 도구.

## 주요 기능

### 1. 회사 검색
- 회사명을 입력받아 DART `corpCode.xml`(전체 회사 고유번호 목록)에서 매칭
- 완전일치 우선, 실패 시 부분일치 후보 목록 제시 → 사용자가 번호로 선택
- 상장사(종목코드 존재)를 후보 정렬 시 우선 노출
- corpCode.xml은 로컬에 캐싱(7일 주기 갱신)하여 매 실행마다 재다운로드하지 않음

### 2. 재무제표 조회
- 대상: 연결재무제표(CFS), 사업보고서(연간, reprt_code=11011)
- 비교 기간: 최근 3개년 (기본값 자동 계산, `--years` 옵션으로 override 가능)
- API: DART `fnlttSinglAcntAll.json` (전체 재무제표)

### 3. 비교 항목
| 구분 | 항목 |
|---|---|
| 재무상태표 | 자산총계, 부채총계, 자본총계 |
| 손익계산서 | 매출액, 영업이익, 당기순이익 |
| 현금흐름표 | 영업활동현금흐름, 투자활동현금흐름, 재무활동현금흐름 |
| 재무비율 (자동계산) | ROE(%), 부채비율(%) |

- 계정 매칭은 XBRL 표준 계정ID(`account_id`)를 1순위로, 실패 시 계정명 별칭(alias)으로 2차 매칭
- 매칭 실패/데이터 없음 항목은 "N/A"로 표기하고 실행 종료 시 경고 목록 출력

### 4. 엑셀 출력
- 파일명: `output/재무분석_YYYYMMDD_HHMMSS.xlsx`
- 구성: 비교요약 시트 1개(연도별로 3개사 나란히 비교) + 회사별 상세 시트 3개
- 서식: 헤더 강조, 금액 천단위 구분, 비율 소수점 표기, 열 너비 자동조정, 헤더 행 고정

## 비목표 (Out of Scope)

- 별도재무제표, 분기/반기 보고서 조회
- 4개 이상 회사 동시 비교, GUI/웹 인터페이스
- 다중 사용자 지원, 클라우드 배포
- 재무제표 이외의 공시정보(지분공시, 대량보유 등) 조회

## 기술 스택

Python 3.14 / `requests`(API 호출), `openpyxl`(엑셀 생성), `python-dotenv`(API 키 관리). 클래스 계층 없이 함수 위주의 평면 모듈 구조 (`config.py`, `corp_code.py`, `dart_client.py`, `account_map.py`, `excel_writer.py`, `main.py`).

## 제약사항 / 알려진 한계

- 기본 연도 계산은 12월 결산법인 기준 (다른 결산월 회사는 `--years`로 직접 지정)
- 금융업·지주사 등 특수 계정체계를 쓰는 회사는 일부 항목이 매칭되지 않을 수 있음
- DART API 요청 제한(트래픽 제한) 고려해 재시도/백오프 로직 적용, 대량 반복 실행은 비권장

## 성공 기준

- 실제 회사 3곳(예: 삼성전자, SK하이닉스, LG전자)에 대해 경고 없이 9개 항목 + 2개 비율이 정확히 채워진 엑셀 파일이 생성됨 — 검증 완료
- 존재하지 않는 회사명 입력 시 재입력 유도, 데이터 없는 연도는 예외 없이 "N/A" 처리 — 검증 완료
